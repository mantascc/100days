Working with data as raw material. Low-level primitives that carry complexity without decorating it. Every element earns its presence by transmitting meaning. The aesthetic is a byproduct of function — what remains when noise is removed.

§01
principles

Five rules that hold. When in doubt, apply them in order — each one removes something.

01
quiet systems over loud interfaces
The interface recedes. Data is foregrounded. Decoration signals insecurity — a system that works has nothing to prove.
02
visible logic, emergent complexity
Rules are simple. Behaviour is complex. The viewer should be able to infer the rule from watching — then be surprised by what it generates.
03
reduction as polish
Strip, don't add. Every element must earn its presence. If removing it doesn't break meaning, it was noise.
04
data as material — show the grain
Don't smooth over the texture of the data. Variance, drift, error, boundary — these are the material. Let them show.
05
physics over easing, linear over theatrical
Euler integration beats CSS transitions. Things drift and settle. They do not bounce. They do not spring. They move because forces act on them.
06
one accent, total discipline
Pick a single high-contrast accent per project. Use it exactly where meaning needs to land. Everywhere it appears is a claim — so make the claim count.
§02
color system

§03
typography

Monospace for everything. Instrument Serif for display moments only — hero titles, pull quotes, editorial. All lowercase OR all uppercase. Never mixed.

§04
motion & primitives

RequestAnimationFrame. Linear interpolation. No springs, no bounces. Circles, lines, rectangles — no organic curves unless algorithmically generated. Density creates form.

§05
anti-patterns

The system is defined as much by what it refuses as by what it includes.

✗drop shadows
