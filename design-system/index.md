# Design System — index

Start at the [README](README.md) for the model and the status lifecycle.

## Principles (tensions)

- [quiet-over-loud](principles/quiet-over-loud.md) · active v1.0 — restraint vs presence; recede so the system can be seen.
- [visible-logic](principles/visible-logic.md) · active v1.0 — legibility of mechanism vs richness of result; show the rule.
- [reduction-as-polish](principles/reduction-as-polish.md) · active v1.0 — stripping vs expressiveness; *under pressure, 2 exceptions.*
- [data-as-material](principles/data-as-material.md) · active v1.0 — raw fidelity vs composed clarity; show the grain.
- [physics-over-easing](principles/physics-over-easing.md) · active v1.0 — real integration vs theatrical timing; simulate, don't choreograph.
