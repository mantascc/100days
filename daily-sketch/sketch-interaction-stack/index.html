Field Notes Frontend / Agentic UI Replaces: design-essentials@2014

The unit of design
moved from library
to contract

A decade ago, design essentials meant Bootstrap, Foundation, a grid system, a CSS reset, a sprite of icons. Frameworks that decided for you. Today the substrate is composable, copy-pasted, and increasingly legible to systems that didn't write it. The list below is a snapshot of what replaced the list.

// what an interaction primitive looks like in 2026
{
  "id": "button.primary",
  "behavior": "radix.button",  // headless contract
  "appearance": "tailwind.tokens", // utility substrate
  "distribution": "shadcn.registry", // copy, don't install
  "motion": "motion.spring",
  "agent_readable": true
}
§ 01

The shift in units of abstraction

Each generation of frontend tooling has made the substrate more composable and more legible to systems that didn't author it. The trajectory matters more than any single library on it.

Framework. Bootstrap, Foundation. A finished product. The button is decided. You inherit a worldview along with the CSS — opinions about grids, breakpoints, typography. Shipping speed in exchange for design surrender.
§ 02

The current stack

Nine layers. Each one is a contract, not a product. The interesting question for any new tool is no longer "what does it look like" but "what does it expose."

L01 / BEHAVIORheadless
Headless primitives
behavior decoupled from appearance

Radix, React Aria, Ark UI, Base UI. The component is a contract — keyboard handling, ARIA, focus management — without a single line of styling. You bring the look. The library brings the correctness.

a11y keyboard focus aria
L02 / DISTRIBUTIONcopy-paste
Registry over package
code you own, not code you depend on

shadcn/ui broke the npm install model. Components live inside your repo. You read them, edit them, fork them. The library is a registry of recipes — the contract is "here is good code, take it."

ownership forkable recipe
L03 / STYLINGutility
Tailwind substrate
styling as configuration

Utility-first won. CSS-in-JS receded into legacy. Tailwind v4 collapsed config into CSS itself. Most new component systems now assume Tailwind underneath — it became the styling lingua franca by being boring.

tokens atomic jit
L04 / MOTIONspring
Motion as legibility
animation that explains state

Motion (née Framer Motion), Motion One, GSAP. Animation stopped being decoration and became a way to make state transitions legible. Spring physics replaced easing curves. Time became a design surface.

physics spring gesture
L05 / PATTERNsingle-purpose
Pattern primitives
one library, one interaction

Vaul for drawers. cmdk for command palettes. sonner for toasts. react-resizable-panels. Each library does exactly one thing and refuses to do anything else. The Emil Kowalski axis — small, surgical, opinionated about correctness only.

surgical opinionated composable
L06 / CANVASspatial
Beyond the DOM
when HTML hits the ceiling

tldraw, Excalidraw, Rive, Lottie. Interfaces that aren't pages — infinite canvases, spatial UIs, runtime-animated vector. Where you go when the artifact is a graph topology, an agent trace, or anything DOM was never asked to do.

canvas spatial runtime-vector
L07 / 3Dwebgpu
3D in product surface
no longer reserved for marketing

React Three Fiber, Theatre.js, Spline. WebGPU shaders showing up in product UI, not just hero sections. The boundary between "interface" and "scene" softens. Still niche, still loud — but no longer optional in some categories.

r3f webgpu shader
L08 / GENERATIVEai-native
Generative UI
interfaces composed at runtime

Two camps. Reactive: v0, Vercel AI SDK UI, Thesys — intent collapses directly to component. Compositional: A2UI vocabularies, LangGraph state-driven UI — interfaces assembled from a catalog of agent-readable parts.

reactive compositional a2ui v0
L09 / SCHEMAhandoff
Design tokens as grammar
the lingua franca of handoff

Style Dictionary. The W3C design tokens spec stabilizing. Tokens became the contract between Figma, code, and increasingly, agents — the smallest unit of design intent that any system can read and write.

w3c-tokens json agent-readable
